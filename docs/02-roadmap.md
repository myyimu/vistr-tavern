# Roadmap

## Phase 0: Private Prototype

Goal: Prove that human intrusion inside an AI ensemble creates useful dramatic tension.

Deliverables:

- SillyTavern extension shell.
- Single-character takeover.
- Time-limited AI recovery.
- Basic message logging.
- Basic Markdown export.

Success criteria:

- A user can create one scene where a human intrusion changes the story direction.
- The exported log contains useful story material.

Open source status:

- Keep private.

## Phase 1: Usable Extension

Goal: Make the core loop usable by early testers.

Deliverables:

- Multi-character takeover support.
- Anonymous intrusion mode.
- Immersion mode and god mode.
- Scene management.
- Tension tracking.
- Intrusion lifecycle log.
- JSON and Markdown export.
- Stable local schema.

Success criteria:

- A tester can install the extension and run a complete intrusion scene without developer help.
- The exported material clearly shows what changed because of the human anomaly.

Open source status:

- Closed alpha or limited private testing.

## Phase 2: Narrative Structuring Tool

Goal: Move from raw logs to structured creative material.

Deliverables:

- Key event extraction.
- Relationship change extraction.
- World-state change tracking.
- Timeline export.
- LLM prompt pack export.
- SQLite local archive.
- Example story sessions.

Success criteria:

- Writers can reuse exported material for outlines, chapters, tabletop logs, or OC development.
- VistrTavern has a recognizable schema and workflow.

Open source status:

- Recommended public open source point.

## Phase 3: Creator Workspace

Goal: Manage long-running story projects.

Deliverables:

- Character archive.
- World archive.
- Scene library.
- Relationship graph.
- Session history.
- Timeline view.
- Chapter material organizer.
- Multi-session project archive.

Success criteria:

- VistrTavern can support an ongoing story, not only one-off scenes.

Open source status:

- Public repository with contribution guidelines.

## Phase 4: Multiplayer Intrusion

Goal: Make real visitor intrusion a networked experience.

Deliverables:

- WebSocket backend.
- Room/session model.
- Host permissions.
- Takeover request workflow.
- Visitor queue.
- Delayed identity reveal.
- Audit log.
- PostgreSQL persistence.

Success criteria:

- Multiple users can participate in one AI ensemble session.
- Human intrusion can be moderated without breaking immersion.

## Phase 5: Open Narrative Platform

Goal: Build an extensible ecosystem around AI ensemble disturbance.

Deliverables:

- Plugin API.
- Disturbance rule packs.
- Role packs.
- World templates.
- Replay system.
- Narrative analytics.
- Community examples.
- Optional hosted services.

Success criteria:

- Other creators can build workflows, rules, and templates on top of VistrTavern.
