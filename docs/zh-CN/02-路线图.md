# 路线图

## Phase 0：私有原型

目标：证明真人混入 AI 群像后确实能产生有价值的戏剧张力。

交付物：

- SillyTavern 插件空壳。
- 单角色接管。
- 限时恢复 AI。
- 基础消息记录。
- 基础 Markdown 导出。

成功标准：

- 用户能创建一个真人入侵改变剧情方向的场景。
- 导出的记录包含可用的故事素材。

开源状态：

- 保持私有。

## Phase 1：可用插件

目标：让核心闭环可以被早期测试者使用。

交付物：

- 多角色接管支持。
- 匿名入侵模式。
- 沉浸模式和上帝模式。
- 场景管理。
- 张力追踪。
- 入侵生命周期日志。
- JSON 和 Markdown 导出。
- 稳定本地 schema。

成功标准：

- 测试者可以在没有开发者协助的情况下安装插件并完成一次入侵场景。
- 导出内容能清楚展示“真人异常源”造成了什么变化。

开源状态：

- 封闭 alpha 或小范围私测。

## Phase 2：叙事结构化工具

目标：从原始记录升级为结构化创作素材。

交付物：

- 关键事件提取。
- 人物关系变化提取。
- 世界状态变化追踪。
- 时间线导出。
- LLM Prompt 包导出。
- SQLite 本地档案。
- 示例故事会话。

成功标准：

- 作者可以把导出素材用于大纲、章节、跑团日志或 OC 发展。
- VistrTavern 拥有可识别的 schema 和工作流。

开源状态：

- 推荐在这个阶段正式公开开源。

## Phase 3：创作者工作台

目标：管理长期故事工程。

交付物：

- 角色档案。
- 世界观档案。
- 场景库。
- 人物关系图。
- 会话历史。
- 时间线视图。
- 章节素材整理器。
- 多会话项目归档。

成功标准：

- VistrTavern 可以支持持续创作，而不只是一次性场景。

开源状态：

- 公开仓库，并配套贡献指南。

## Phase 4：多人入侵

目标：让真实 Visitor Intrusion 成为网络化体验。

交付物：

- WebSocket 后端。
- 房间 / 会话模型。
- 房主权限。
- 接管申请流程。
- 访客队列。
- 延迟身份揭示。
- 审计日志。
- PostgreSQL 持久化。

成功标准：

- 多个用户可以参与同一个 AI 群像会话。
- 真人入侵可以被管理，同时不破坏沉浸感。

## Phase 5：开放叙事平台

目标：围绕 AI 群像扰动建立可扩展生态。

交付物：

- 插件 API。
- 扰动规则包。
- 角色包。
- 世界观模板。
- Replay 回放系统。
- 叙事分析。
- 社区示例。
- 可选托管服务。

成功标准：

- 其他创作者可以基于 VistrTavern 构建自己的规则、模板和工作流。
