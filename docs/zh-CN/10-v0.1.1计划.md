# v0.1.1 小版本计划

`v0.1.1` 是 public alpha 的稳定性小修版本。不要加入后端服务、账号系统、多人支持或大型 UI 重构。

状态：已作为 `v0.1.1-alpha` 实现。

## 目标

- 让 handoff 注入失败更容易诊断。
- 让 Debug 面板对测试者更有用。
- 记录第一批确认可用的 SillyTavern 版本。
- 降低手动安装支持成本。

## 候选改动

- pending handoff 存在但 `generate_interceptor` 一直没有被调用时，显示可见提示。
- 增加一键复制完整 debug snapshot。
- 在使用手册里解释 `lastInterceptorCallAt` 和 `lastInjectionResult`。
- 在 README 增加公开验证用的已测试版本表。
- 补充 `VT` 面板、Debug 面板、handoff pending 状态和导出结果截图占位。
- 如果 AI 回复过度元叙事或太冗长，收紧 handoff prompt 文案。

## 非目标

- 不做新后端。
- 不做账号系统。
- 不引入数据库。
- 不做扩展市场打包。
- 不做多角色实时接管。
- 不做大型视觉重构。

## 发布标准

- `npm run smoke` 通过。
- `node --check index.js core/narrativeMemory.js core/exportWriter.js ui/uiOverlay.js` 通过。
- 从公开 GitHub 仓库做一次干净手动安装并成功运行。
- 至少记录一个真实测试过的 SillyTavern 版本。
- 至少处理一个 alpha feedback issue，或转成明确的后续 issue。
