# 发布检查清单

发布 `v0.3.0-alpha` 前使用这份清单。

## 仓库

- [ ] 仓库名为 `vistr-tavern`。
- [ ] 仓库可见性为 public。
- [ ] `manifest.json` 的 `homePage` 指向 `https://github.com/xiaoke5211-star/vistr-tavern`。
- [ ] `package.json`、`manifest.json` 和 `data/version.js` 版本均为 `0.3.0-alpha`。
- [ ] `README.md`、`README.zh-CN.md` 和使用手册都说明这是手动安装版本。

## 本地验证

- [ ] 运行 `npm run verify`。
- [ ] 确认 Markdown 导出包含入侵时间线、真人发言、AI 反应和 continuity handoff。
- [ ] 启用异常察觉后，确认 Markdown 导出包含 `AI 异常察觉` 区块。
- [ ] 标记剧情分支后，确认 Markdown 导出包含 branch points。
- [ ] 确认 Creator Pack 导出包含异常发言、AI 反应、冲突钩子、分支路线和 handoff 上下文。
- [ ] 确认 Character Prompt 导出包含人设提取说明和已记录素材。
- [ ] 确认 JSON 导出包含 `intrusions`、`messages`、`handoffs` 和 `branchPoints`。
- [ ] 运行 `npm run package:zip`，确认生成 `dist/vistr-tavern-0.3.0-alpha.zip`。

## SillyTavern 手动安装

- [ ] 安装目录为 `data/default-user/extensions/vistr-tavern`。
- [ ] 没有重复的 `VistrTavern` 扩展目录。
- [ ] SillyTavern 加载扩展时没有控制台错误。
- [ ] 聊天页面出现浮动 `VT` 按钮。
- [ ] 角色选择器能加载角色。
- [ ] `Debug` 面板显示版本和存储模式。
- [ ] `Debug` 面板显示 compatibility 状态。
- [ ] 在新浏览器状态或清除 `vistr-tavern:first-run-dismissed` 后，首次使用引导会出现。

## 核心闭环

- [ ] 对一个角色开始 intrusion。
- [ ] 记录真人异常发言。
- [ ] intrusion 激活期间让 AI 回复。
- [ ] 手动或等待超时结束 intrusion。
- [ ] 确认生成 continuity handoff。
- [ ] 确认 pending handoff 数量变化。
- [ ] 触发下一次生成。
- [ ] 确认 Debug 中能看到 interceptor 调用。
- [ ] 确认 AI 回复后 handoff 变为 consumed。
- [ ] 确认 handoff 没有作为普通聊天记录永久保存。
- [ ] 选择 `断片` + `Controlled character`，确认 handoff 要求生成简短斜体内心独白。
- [ ] 选择 `怀疑` + `Both`，确认 self 和 observer awareness events 会被导出。

## Fallback 和导出

- [ ] `Copy Latest Handoff` 可以复制 handoff 文本。
- [ ] Markdown 导出成功。
- [ ] JSON 导出成功。
- [ ] Creator Pack 导出成功。
- [ ] Character Prompt 导出成功。
- [ ] 导出内容没有超出本地会话故事素材之外的意外隐私数据。
