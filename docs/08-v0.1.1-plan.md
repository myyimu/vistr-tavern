# v0.1.1 Patch Plan

`v0.1.1` should be a small public-alpha stabilization release. Do not add backend services, accounts, multi-user support, or a large UI redesign.

## Goals

- Make failed handoff injection easier to diagnose.
- Make the Debug panel more useful for testers.
- Document the first confirmed SillyTavern versions.
- Reduce support friction for manual installation.

## Candidate Changes

- Show a visible warning when `generate_interceptor` has not been called after a pending handoff exists.
- Add a one-click copy action for the full debug snapshot.
- Add `lastInterceptorCallAt` and `lastInjectionResult` explanations to the user guide.
- Add a tested-version table to README after real validation.
- Add screenshots for the `VT` panel, Debug panel, handoff pending state, and export result.
- Tighten wording in the handoff prompt if AI replies become too meta or too verbose.

## Non-Goals

- No new backend.
- No account system.
- No database.
- No marketplace packaging.
- No multi-character live takeover.
- No major visual redesign.

## Release Criteria

- `npm run smoke` passes.
- `node --check index.js core/narrativeMemory.js core/exportWriter.js ui/uiOverlay.js` passes.
- Clean manual install works from the public GitHub repository.
- At least one real SillyTavern version is listed as tested.
- One alpha feedback issue has been reviewed and either addressed or converted into a follow-up issue.
